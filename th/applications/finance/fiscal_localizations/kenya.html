การกำหนดค่า¶

การติดตั้ง 🇰🇪 ** เคนยา**: การอ้างอิง: `โปรแกรมการแปลทางการเงิน<fiscal_locations/packages>`รับคุณสมบัติทั้งหมดในเคนยาแปลเป็นภาษาท้องถิ่น

eTIMS¶

สำนักงานภาษีเคนยา (KRA) https://www.kra.go.ke/ > `_ มีใช้ระบบการจัดการใบกำกับภาษีอิเล็กทรอนิกส์ (eTIMS)< https://www.kra.go.ke/online-services/etims > `_ สำหรับการจัดเก็บภาษี

ในการยื่นเอกสารผ่าน eTIMS คุณต้องใช้: abbr:`OSCU (ขายออนไลน์หน่วยควบคุม) บูรณาการกับระบบใบแจ้งหนี้เทรดเดอร์ที่มีอยู่ (มอก.),ยกตัวอย่างจาก โอดู OSCU สำหรับการตรวจสอบ,เข้ารหัส ลงนาม โอน และจัดเก็บใบกำกับภาษี

Note

โปรดตรวจสอบให้แน่ใจว่า:`install<general/install>`eTIMS EDI ในเคนยา**โมดูลใช้อุปกรณ์ OSCU อย่างเต็มที่

เริ่มต้นอุปกรณ์ OSCU¶

OSCU ต้องเริ่มใช้งานก่อนใช้งาน สำหรับสิ่งนี้โปรดไปที่: menuselection:`setting--> general setting', click:guilabel:`updateป้อน "ข้อมูล" ในส่วน: guilabel: "บริษัท" และใส่ของคุณ: guilabel: "ภาษี"ID。

การเริ่มต้น OSCU:

  1. ไปที่: menusesection:`setting-->general settings', เลื่อนลงไปที่: guilabel: "การบูรณาการ eTIMS ในเคนยา" ส่วน

  2. ตั้งค่า: guilabel: "โหมดเซิร์ฟเวอร์ eTIMS" เป็น: guilabel: "ทดสอบ"การเริ่มต้น

  3. ป้อนอุปกรณ์ของคุณ: guilabel: "หมายเลขซีเรียล" ทำเครื่องหมายสองช่องกล่อง

  4. ฮิต: guilabel:`เริ่มต้น OSCU'

Note

มีสามโหมดเซิร์ฟเวอร์ให้เลือก:

  • : guilabel: "การสาธิต": ออกแบบมาเพื่อการสาธิต; มันใช้ข้อมูลอะนาล็อกแทนจำเป็นต้องเริ่มต้น OSCU;

  • : guilabel: test: สำหรับการทดสอบการเชื่อมต่อกับ eTIMS;

  • : guilabel: Production: ฐานข้อมูลแบบเรียลไทม์สำหรับการเตรียมส่งข้อมูล

Important

หากอุปกรณ์ของคุณมี ** เริ่มต้นใช้งาน ** (ผ่าน ERP อื่นตัวอย่างเช่น), เปิดใช้งาน: doc:`... // โหมดทั่วไป / นักพัฒนา จากนั้น, ใน: guilabel: "การบูรณาการ eTIMS ในเคนยา" ใน: guilabel: ฟิลด์ "unit ID" และก่อนผ่านเริ่มต้นในฟิลด์: guilabel: "CMC Key" เมื่อคลิก: guilabel: "บันทึก"เรียบร้อย

เมื่อ ** โมดูล OSCU ** ถูก: อ้าง:`Initialization <Kenya / Initialization>',สร้างหมายเลขซีเรียล OSCU สำหรับแต่ละ บริษัท ในฐานข้อมูลนี้ของ ** ประเทศ ** จะเปลี่ยนเป็น: guilabel: "เคนยา". สร้างหมายเลขซีเรียลแล้วขึ้นอยู่กับหมายเลขภาษีมูลค่าเพิ่มของบริษัท (โดยไม่คำนึงถึงความถูกต้อง) นี่คือหมายเลขซีเรียลต่อเนื่องที่ไม่ซ้ำกันที่ขึ้นต้นด้วยคำนำหน้า "ODOO"ตามด้วยหมายเลขภาษีมูลค่าเพิ่มของบริษัท

Important

ทั้งหมด ** การเคลื่อนไหวของหุ้น ** ต้องส่งให้ KRA พวกเขาไม่ต้องการออกใบแจ้งหนี้หากเป็นการดำเนินงานภายในหรือการปรับสินค้าคงคลัง ดังนั้นOdoo จะส่งให้โดยอัตโนมัติหากมีอย่างน้อยหนึ่งอย่างต่อไปนี้ความพึงพอใจ:

  1. ย้ายรายชื่อที่ไม่ได้ตั้งค่าไว้

  2. ผู้ติดต่อคือ บริษัท หลักหรือสาขาของ บริษัท หลักของคุณ

Note

  • ต้องยืนยันการเปลี่ยนแปลงสินค้าคงคลังก่อนส่งใบแจ้งหนี้ไปยัง eTIMS

  2. Odoo View: guilabel: "หมายเลขภาษี" (PIN) สำหรับผู้ขาย (คู่ค้า);

    • สร้างผู้ติดต่อใหม่ (หุ้นส่วน) หากไม่รู้จัก

    • Odoo จะใช้ผู้ติดต่อที่รู้จักหากทราบและรหัสสาขาเดียวกัน

  3. ในการเรียกเก็บเงินที่ถอนจาก KRA เลือก: guilabel: "ผลิตภัณฑ์" แต่ละใบเรียกเก็บเงินผู้ขาย * ต้อง * รวมผลิตภัณฑ์ที่รอการยืนยันและส่งไปยัง eTIMS ในภายหลังขึ้น

  4. Odoo ตรวจสอบบรรทัดใบสั่งซื้อที่มีอยู่ให้ตรงกับผลิตภัณฑ์ที่ป้อนขั้นตอนก่อนหน้าและพันธมิตรถ้ามี ฮิต: แท็ก: `Buyฟิลด์ "บรรทัดคำสั่งซื้อ" และเลือกบรรทัดใบสั่งซื้อที่เกี่ยวข้องที่ถูกต้องจับคู่ผลิตภัณฑ์ จำนวนในใบเรียกเก็บเงิน * ต้อง * ด้วยจำนวนที่ระบุไว้ในใบสั่งซื้อ

  2. ไปที่: ส่วนเมนู: `การตั้งค่า -> เทคโนโลยี -> ระบบอัตโนมัติ: แผนการกระทำและการค้นหา: guilabel: KE eTIMS: รับการนำเข้าศุลกากรจากประเทศOSCU。

  3. คลิกการกระทำในรายการและคลิก: guilabel: "เรียกใช้ด้วยตนเอง" เพื่อรับรหัส

ไปที่: ส่วนเมนู: "บัญชี -> ผู้จำหน่าย -> ศุลกากรนำเข้า" ดูรหัสที่ใส่

ส่งและลงลายมือชื่อ** ศุลกากรนำเข้า** ขั้นตอนต่อไปนี้KRA：

  1. ไปที่: เมนู: "บัญชี -> ซัพพลายเออร์ -> ศุลกากรนำเข้า"; นี่การนำเข้าทางศุลกากรจะถูกสกัดจาก KRA โดยอัตโนมัติ

  2. จับคู่รายการที่นำเข้ากับผลิตภัณฑ์ที่จดทะเบียนแล้วใน: guilabel: ฟิลด์ "ผลิตภัณฑ์" (สร้างผลิตภัณฑ์หากไม่มีผลิตภัณฑ์ที่เกี่ยวข้องการมีอยู่)

  3. ตั้งค่าซัพพลายเออร์ใน: guilabel: "พาร์ทเนอร์" ฟิลด์

  4. จับคู่สินค้าที่นำเข้ากับการซื้อที่เกี่ยวข้องตามคู่ค้าการสั่งซื้อ (ดู: อ้าง: `ขั้นตอนการซื้อ <Kenya / Buy> `) สินค้าคงคลังต้องปรับถูกต้องเมื่อศุลกากรอนุมัติให้นำเข้า

    หากไม่มีใบสั่งซื้อที่เกี่ยวข้องให้สร้างหนึ่งใบแล้ว: guilabel: "ยืนยัน"จากนั้นคลิก: guilabel: "รับสินค้า" เพื่อยืนยันการจัดส่ง: guilabel: "ยืนยัน" ในใบสั่งซื้อ

  5. คลิก: guilabel: "การจับคู่และการอนุมัติ" หรือ: guilabel: "การจับคู่และการปฏิเสธ",ขึ้นอยู่กับสภาพของสินค้า

Note

ไฟล์ JSON ที่ได้รับจาก KRA แนบไปกับประวัติการแชทศุลกากรนำเข้า

BOM¶

KRA ขอให้ส่ง BOM ทั้งหมดให้พวกเขา การส่ง BOM ไปยัง eTIMSผลิตภัณฑ์และส่วนประกอบ * ต้อง * เป็น: อ้าง:` ลงทะเบียน <Kenya / ผลิตภัณฑ์ -ลงทะเบียน> `. หากต้องการเข้าถึงรายการวัสดุของผลิตภัณฑ์ให้คลิกที่ผลิตภัณฑ์และจากนั้นคลิกที่: guilabel: "รายการวัสดุ" ปุ่มอัจฉริยะ

ตรวจสอบให้แน่ใจว่า: อ้าง: ฟิลด์ที่จำเป็นสำหรับ `KRA <เคนยา / การลงทะเบียนผลิตภัณฑ์> สำหรับกรอกข้อมูล: guilabel: ส่วน "รายละเอียด KRA eTIMS": guilabel: แท็บ "บัญชี" ในแบบฟอร์มผลิตภัณฑ์และคลิก: guilabel: `ส่งไปที่eTIMS。 ยืนยันการส่ง BOM สำเร็จในการแชทคุณสามารถค้นหาข้อมูลที่ส่งมาในไฟล์ JSON ที่แนบมาด้วย

ตั๋วเครดิต¶

KRA ไม่ยอมรับปริมาณหรือราคาสูงกว่าใบแจ้งหนี้เริ่มต้น เมื่อสร้างสลิปเครดิตเหตุผล KRA ต้องเป็นหมายเหตุ: ในแบบฟอร์มเครดิตสลิปไปที่: guilabel: แท็บรายละเอียด eTIMS,เลือก: guilabel: `eTIMS Credit Slip Reason' จากนั้นเลือกใน: guilabel: หมายเลขใบแจ้งหนี้ในฟิลด์ "ย้อนกลับ"
