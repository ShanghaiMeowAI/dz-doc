ข้อมูลบริษัท¶

การกำหนดค่าข้อมูลของบริษัททำให้มั่นใจได้ว่าฐานข้อมูลระบบบัญชีของคุณได้รับการตั้งค่าอย่างเหมาะสม หากต้องการเพิ่มข้อมูล ให้ไปที่ การตั้งค่า -> การตั้งค่าทั่วไป และในส่วน บริษัท ให้คลิก อัปเดตข้อมูล จากที่นี่ ให้กรอกข้อมูลในฟิลด์:

  • ที่อยู่: ที่อยู่ของบริษัท;

  • ภาษี: ภาษีมูลค่าเพิ่มของบริษัท;

  • Codice Fiscale: รหัสทางการเงินของบริษัท;

  • ระบบภาษี: ระบบภาษีที่บริษัทอยู่ภายใต้;

การกำหนดค่าภาษี¶

ฟีเจอร์การออกใบแจ้งหนี้อิเล็กทรอนิกส์หลายอย่างมีการใช้งานโดยใช้ระบบภาษีของ Odoo ด้วยเหตุนี้ ภาษีจึงต้องได้รับการกำหนดค่าอย่างเหมาะสมเพื่อสร้างใบแจ้งหนี้อย่างถูกต้อง และจัดการกรณีการใช้การเรียกเก็บเงินอื่นๆ

การประยุกต์ใช้สำหรับประเทศ อิตาลี มี ตัวอย่าง ภาษีที่กำหนดไว้ล่วงหน้าเพื่อวัตถุประสงค์ต่างๆ

การยกเว้นภาษี¶

หน่วยงานของประเทศอิตาลีกำหนดให้ใช้ภาษีการขายจำนวน ศูนย์เปอร์เซ็นต์ (0%) เพื่อติดตาม ประเภทการยกเว้นภาษี (ธรรมชาติ) และ การอ้างอิงกฎหมาย ที่ถูกต้อง การยกเว้นที่ดำเนินการในบรรทัดใบแจ้งหนี้

Example

ภาษีส่งออกในสหภาพยุโรปสามารถใช้เป็นข้อมูลอ้างอิงได้ (0% EU, ป้ายกำกับใบแจ้งหนี้ 00eu) สามารถพบได้ภายใต้ ระบบบัญชี ‣ การกำหนดค่า ‣ ภาษี การส่งออกได้รับการยกเว้นภาษีมูลค่าเพิ่ม ดังนั้นจึงต้องกรอก การพ้นโทษ และ การอ้างอิงกฎหมาย

See also

มีรหัส ประเภทการยกเว้นภาษี (ธรรมชาติ) และ การอ้างอิงกฎหมาย มากมาย ตรวจสอบให้แน่ใจว่าคุณตรวจสอบเวอร์ชันล่าสุดที่มีอยู่เพื่อรับข้อมูลล่าสุดเกี่ยวกับ:

  • เอกสารของทางการประเทศอิตาลี

  • คู่มืออย่างเป็นทางการเกี่ยวกับการยกเว้นภาษี

Note

หากคุณต้องการใช้การแก้ต่างประเภทอื่น ให้ไปที่ ระบบบัญชี ‣ การกำหนดค่า ‣ ภาษี เลือกภาษีที่คล้ายกัน จากนั้นคลิกที่ไอคอนฟันเฟือง และเลือก คัดลอก ในแท็บ ตัวเลือกขั้นสูง ให้เพิ่ม การพ้นโทษ และ การอ้างอิงกฎหมาย เพื่อยืนยันคลิกที่ บันทึก

Tip

เปลี่ยนชื่อ ภาษีของคุณในช่อง ชื่อ ตาม การพ้นโทษ เพื่อแยกความแตกต่างได้อย่างง่ายดาย

การเรียกเก็บเงินย้อนกลับ¶

กลไก การเรียกเก็บภาษีย้อนหลัง คือกฎ VAT ที่เปลี่ยนความรับผิดในการชำระ VAT จากซัพพลายเออร์ไปยังลูกค้า ลูกค้าชำระ VAT ด้วยตนเอง ให้กับ AdE แทน มีหลายประเภท:

  • การเรียกเก็บเงินย้อนกลับภายใน (สำหรับการขายในประเทศ)
    ความรับผิดชอบด้านภาษีมูลค่าเพิ่มจะตกเป็นของผู้ซื้อสำหรับผลิตภัณฑ์และบริการบางประเภท
  • การเรียกเก็บเงินย้อนกลับภายนอก (สำหรับการขายภายในสหภาพยุโรป)
    ภาษีมูลค่าเพิ่มจะครบกำหนดในประเทศที่จัดส่งหรือในประเทศที่ให้บริการ เมื่อผู้ซื้อเป็นธุรกิจในประเทศอิตาลี สหภาพยุโรปจะเสนอกลไกที่ช่วยให้ผู้ขายสามารถโอนความรับผิดชอบของเขาไปยังผู้ซื้อได้

การแจ้งหนี้¶

ใบแจ้งหนี้ของลูกค้าที่เรียกเก็บเงินแบบย้อนกลับ จะไม่แสดงจำนวน VAT แต่ AdE กำหนดให้ผู้ขายระบุเหตุผล การยกเว้นภาษี และ การอ้างอิงกฎหมาย ที่ทำให้กลไกการเรียกเก็บเงินแบบย้อนกลับเกิดขึ้นได้ Odoo จัดเตรียมชุดภาษีพิเศษ 0% ที่สามารถกำหนดให้กับบรรทัดใบแจ้งหนี้ที่เรียกเก็บย้อนกลับแต่ละบรรทัด ซึ่งแสดงถึงการกำหนดค่าที่ใช้บ่อยที่สุด

ใบเรียกเก็บเงินผู้ขาย¶

บริษัทอิตาลีที่ถูกเรียกเก็บค่าธรรมเนียมย้อนกลับจะต้องส่งข้อมูลในใบเรียกเก็บเงินที่ได้รับไปที่ AdE

Note

ไฟล์ VAT XML ที่รายงานด้วยตนเองจะต้องออกและส่งไปที่ AdE สำหรับการเรียกเก็บเงินแบบย้อนกลับ

เมื่อสร้างใบเรียกเก็บเงินของผู้ขาย คุณสามารถเพิ่มภาษี ค่าธรรมเนียมย้อนกลับ ได้ในฟิลด์ ภาษี คุณสามารถตรวจสอบภาษีที่มีอยู่ได้โดยไปที่ ระบบบัญชี ‣ การกำหนดค่า ‣ ภาษี คุณจะเห็นว่าภาษีสินค้า 10% และภาษีบริการ 22% เปิดใช้งานอยู่ และอื่นๆ อีกมากมาย เนื่องจากการตั้งค่าคอนฟิกอัตโนมัติของสถานะทางบัญชีของประเทศอิตาลี สิ่งเหล่านี้จึงถูกเปิดใช้งานโดยอัตโนมัติในรายการภาษี

ตารางภาษี¶

การประยุกต์ใช้สำหรับประเทศอิตาลีมีส่วนเฉพาะ ตารางภาษี สำหรับภาษี ค่าธรรมเนียมย้อนกลับ ตารางภาษีเหล่านี้สามารถระบุได้ด้วยแท็ก VJ และสามารถพบได้โดยไปที่ ระบบบัญชี ‣ การรายงาน ‣ รายงานการตรวจสอบ: รายงานภาษี

ใบแจ้งหนี้อิเล็กทรอนิกส์¶

SdI คือระบบ ใบแจ้งหนี้อิเล็กทรอนิกส์ ที่ใช้ในประเทศอิตาลี ช่วยให้สามารถส่งและรับใบแจ้งหนี้อิเล็กทรอนิกส์ไปยังและจากลูกค้าได้ เอกสารต้องอยู่ในรูปแบบ XML EDI ที่เรียกว่า FatturaPA และได้รับการตรวจสอบอย่างเป็นทางการจากระบบก่อนส่งมอบ

เพื่อให้สามารถรับใบแจ้งหนี้และการแจ้งเตือนได้ บริการ SdI จะต้องได้รับการแจ้งว่าไฟล์ของผู้ใช้จะต้องถูกส่งไปยัง Odoo และดำเนินการในนามของพวกเขา ในการทำเช่นนั้น คุณต้องตั้งค่า รหัสปลายทาง ของ Odoo บนพอร์ทัล AdE

  1. ไปที่ พอร์ทัลทางการของประเทศอิตาลี และรับรองความถูกต้อง;

  2. ไปที่ส่วน Fatture e Corrispettivi;

  3. ตั้งค่าผู้ใช้เป็นฝ่ายกฎหมายสำหรับหมายเลข VAT ที่คุณต้องการกำหนดค่าที่อยู่อิเล็กทรอนิกส์

  4. ใน Servizi Disponibili ‣ Fatturazione Elettronica ‣ Registrazione dell'indirizzo telematico dove ricevere tutte le fatture elettroniche ให้ใส่ Odoo's Destination Code K95IV18 และยืนยัน

โหมด EDI และการอนุญาต¶

เนื่องจากไฟล์จะถูกส่งผ่านเซิร์ฟเวอร์ของ Odoo ก่อนที่จะส่งไปยัง SdI หรือได้รับจากฐานข้อมูลของคุณ คุณจึงต้องอนุญาตให้ Odoo ประมวลผลไฟล์ของคุณจากฐานข้อมูลของคุณ โดยไปที่ ระบบบัญชี ‣ การกำหนดค่า ‣ การตั้งค่า ‣ การออกใบแจ้งหนี้เอกสารอิเล็กทรอนิกส์

มีสามโหมดให้เลือก:

  • การสาธิต
    โหมดนี้จำลองสภาพแวดล้อมที่มีการส่งใบแจ้งหนี้ไปยังรัฐบาล ในโหมดนี้ ใบแจ้งหนี้จะต้อง ดาวน์โหลดด้วยตนเอง เป็นไฟล์ XML และอัปโหลดไปยังเว็บไซต์ของ AdE
  • ทดสอบ (ทดลอง)
    โหมดนี้จะส่งใบแจ้งหนี้ไปยังบริการที่ไม่ใช้งานจริง (เช่น ทดสอบ) ซึ่งให้บริการโดย AdE การบันทึกการเปลี่ยนแปลงนี้จะทำให้บริษัททั้งหมดในฐานข้อมูลใช้การกำหนดค่านี้
  • ทางการ
    นี่คือโหมดการใช้งานจริงที่ส่งใบแจ้งหนี้ของคุณโดยตรงไปยัง AdE

เมื่อเลือกโหมดแล้ว คุณต้องยอมรับ ข้อกำหนดและเงื่อนไข โดยทำเครื่องหมาย อนุญาตให้ Odoo ประมวลผลใบแจ้งหนี้ จากนั้น บันทึก ตอนนี้คุณสามารถบันทึกธุรกรรมของคุณในระบบบัญชี Odoo ได้แล้ว

Warning

การเลือก ทดสอบ (ทดลอง) หรือ ทางการ นั้น ไม่สามารถย้อนกลับได้ ตัวอย่างเช่น เมื่ออยู่ใน โหมดอย่างเป็นทางการ จะไม่สามารถเลือก ทดสอบ (ทดลอง) หรือ การสาธิต เราขอแนะนำให้สร้าง ฐานข้อมูลแยกต่างหาก เพื่อการทดสอบเท่านั้น

Note

เมื่ออยู่ใน โหมดทดสอบ (ทดลอง) ใบแจ้งหนี้ทั้งหมดที่ส่ง ต้อง มีคู่ค้าที่ใช้หนึ่งใน รหัสปลายทาง ปลอมต่อไปนี้ ที่กำหนดโดย AdE: 0803HR0 - N8MIMM9 - X9XX79Z การใช้งานจริงใดๆ Codice Destinario ของลูกค้าของคุณจะไม่ได้รับการยอมรับว่าถูกต้องโดยบริการทดสอบ

กระบวนการ¶

การส่งใบแจ้งหนี้ไปยัง SdI สำหรับประเทศอิตาลีเป็นกระบวนการทางอิเล็กทรอนิกส์ที่ใช้สำหรับการส่งเอกสารภาษีในรูปแบบ XML ระหว่างบริษัทและ AdE ไปยัง ลดข้อผิดพลาดและตรวจสอบความถูกต้องของการดำเนินงาน

Note

คุณสามารถตรวจสอบสถานะปัจจุบันของใบแจ้งหนี้ได้ที่ช่อง สถานะ SdI ไฟล์ XML ถูกแนบไปกับใบแจ้งหนี้

การสร้างเอกสาร XML¶

Odoo สร้างไฟล์ XML ที่จำเป็นเป็นสิ่งที่แนบมากับใบแจ้งหนี้ในรูปแบบ FatturaPA ที่ AdE ต้องการ เมื่อเลือกใบแจ้งหนี้ที่ต้องการแล้ว ให้ไปที่ การดำเนินการ และคลิกที่ ส่งและพิมพ์

เมื่อหน้าต่างป๊อปอัปเปิดขึ้น จะมีการดำเนินการให้เลือก สร้างไฟล์ XML จะสร้างไฟล์แนบ

คุณสามารถพบไฟล์ XML และ PDF ที่แนบมากับใบแจ้งหนี้ได้

ยื่นคำร้องต่อ SDI¶

ตัวเลือก ส่งไปที่กรมสรรพากร ในกล่องโต้ตอบ ส่งและพิมพ์ จะส่งไฟล์แนบไปที่ พร็อกซีเซิร์ฟเวอร์ ซึ่งรวบรวมคำขอทั้งหมด จากนั้นส่งต่อผ่านช่องทาง WebServices ไปยัง SdI. ตรวจสอบสถานะการส่งใบแจ้งหนี้ผ่านปุ่ม ตรวจสอบการส่ง ที่ด้านบนของมุมมองของใบแจ้งหนี้

การประมวลผลโดย SDI¶

SdI ได้รับเอกสารและตรวจสอบข้อผิดพลาดใดๆ ในขั้นตอนนี้ ใบแจ้งหนี้จะอยู่ในสถานะ การประมวลผล SDI ดังที่แสดงในใบแจ้งหนี้ ใบแจ้งหนี้ยังได้รับการกำหนดหมายเลข ธุรกรรม FatturaPA ที่แสดงในแท็บ ใบแจ้งหนี้อิเล็กทรอนิกส์ การตรวจสอบอาจใช้เวลาที่เปลี่ยนแปลงได้ ตั้งแต่ไม่กี่วินาทีไปจนถึงหนึ่งวัน ขึ้นอยู่กับคิวของใบแจ้งหนี้ที่ส่งไปทั่วประเทศอิตาลี

การยอมรับ¶

หากเอกสารถูกต้อง เอกสารดังกล่าวจะถูกบันทึกและถือว่าใช้ได้ทางการเงินโดย AdE ซึ่งจะดำเนินการเก็บถาวรใน พื้นที่จัดเก็บทดแทน (Conservazione Sostitutiva) หากมีการร้องขออย่างชัดเจนจาก Agency พอร์ทัล

Warning

Odoo ไม่ได้เสนอข้อกำหนด Conservazione Sostitutiva ผู้ให้บริการรายอื่นและ AdE จัดหาพื้นที่จัดเก็บข้อมูลฟรีและได้รับการรับรองเพื่อให้เป็นไปตามข้อกำหนดที่กฎหมายร้องขอ

SdI รหัสปลายทาง พยายามส่งต่อใบแจ้งหนี้ไปยังลูกค้าตามที่อยู่ที่ให้ไว้ ไม่ว่าจะเป็นที่อยู่อีเมล PEC หรือ SdI รหัสปลายทาง สำหรับช่องทาง WebServices ของ ERP มีการพยายามสูงสุด 6 ครั้งทุกๆ 12 ชั่วโมง ดังนั้นแม้ว่าจะไม่สำเร็จ แต่กระบวนการนี้อาจใช้เวลาถึงสามวัน สถานะใบแจ้งหนี้คือ ยอมรับโดย SDI ส่งต่อไปยังคู่ค้า

การปฏิเสธที่เป็นไปได้¶

SdI อาจพบความไม่ถูกต้องในการคอมไพล์ หรือแม้แต่ข้อมูลที่เป็นทางการก็ได้ ในกรณีนี้ ใบแจ้งหนี้อยู่ในสถานะ SDI ถูกปฏิเสธ ข้อสังเกตของ SdI จะถูกแทรกไว้ที่ด้านบนของแท็บใบแจ้งหนี้ เพื่อแก้ไขปัญหา เพียงลบไฟล์แนบของใบแจ้งหนี้ คืนใบแจ้งหนี้ไปที่ ร่าง และแก้ไขข้อผิดพลาดก็เพียงพอแล้ว เมื่อใบแจ้งหนี้พร้อมแล้ว ก็สามารถส่งใหม่ได้

Note

หากต้องการสร้าง XML ใหม่ ต้องลบทั้งไฟล์แนบ XML และรายงาน PDF เพื่อให้สร้างใหม่พร้อมกัน เพื่อให้แน่ใจว่าทั้งสองมีข้อมูลเดียวกันเสมอ

การส่งต่อเสร็จสมบูรณ์¶

ใบแจ้งหนี้ได้ถูกส่งไปยังลูกค้าแล้ว อย่างไรก็ตาม คุณยังคงสามารถส่งสำเนาให้ลูกค้าในรูปแบบ PDF ผ่านทางอีเมลหรือโพสต์ได้ สถานะคือ ได้รับการยอมรับจาก SDI ส่งมอบให้กับคู่ค้าแล้ว

หาก SdI ไม่สามารถติดต่อกับลูกค้าของคุณได้ แสดงว่าลูกค้าเหล่านั้นอาจไม่ได้ลงทะเบียนในพอร์ทัล AdE ในกรณีนี้ เพียงตรวจสอบให้แน่ใจว่าได้ส่งใบแจ้งหนี้ในรูปแบบ PDF ทางอีเมลหรือทางไปรษณีย์ ใบแจ้งหนี้จะอยู่ในสถานะ ยอมรับโดย SDI การนำส่งของคู่ค้าล้มเหลว

การรวมภาษี¶

เมื่อคุณได้รับใบเรียกเก็บเงินของผู้ขาย จาก SdI จากกระดาษหรือจากไฟล์ XML ที่นำเข้า หน่วยงานภาษีอาจขอให้คุณส่งข้อมูลการรวมข้อมูลภาษีบางส่วนกลับไปที่ SdI ซึ่งจะเกิดขึ้นเมื่อธุรกรรมที่ได้รับการยกเว้นภาษีต้องเสียภาษีไม่ว่าด้วยเหตุผลใดก็ตาม

Example

นี่คือรายการโดยย่อ:

  • การเรียกเก็บเงินย้อนกลับ
    ในฐานะผู้ซื้อ คุณต้องจ่ายภาษีสำหรับสิ่งที่คุณซื้อและรวมข้อมูลภาษี ภาษีย้อนกลับ
  • แยกการชำระเงิน
    ในฐานะผู้ซื้อธุรกิจ PA คุณต้องจ่ายภาษีและรวมข้อมูลภาษี ตรวจสอบให้แน่ใจว่าคุณได้แทนที่ ภาษีการขาย 0% ในใบเรียกเก็บเงินของผู้ขายที่คุณได้รับด้วยภาษี แบ่งการชำระเงิน ที่ถูกต้อง
  • การบริโภคด้วยตนเอง
    ในฐานะเจ้าของธุรกิจ เมื่อคุณใช้สินทรัพย์ที่คุณซื้อเพื่อธุรกิจด้วยเหตุผลส่วนตัวแทน คุณจะต้องจ่ายภาษีที่คุณหักไว้เป็นต้นทุนทางธุรกิจสำหรับสินทรัพย์ดังกล่าว

Odoo อาจตรวจพบว่าใบเรียกเก็บเงินของผู้ขายสามารถตีความได้ว่าเป็นเอกสารประเภทที่ต้องรวมภาษี ตามรายละเอียดในส่วน ประเภทเอกสาร

Important

ตรวจสอบให้แน่ใจว่าคุณได้แทนที่ ภาษีการขาย 0% ในใบเรียกเก็บเงินของผู้ขายที่คุณได้รับด้วยอันที่คุณควรจ่ายให้กับ AdE จากนั้นปุ่มจะปรากฏที่ด้านบนของแบบฟอร์มใบเรียกเก็บเงินของผู้จำหน่ายเดียวเพื่อส่ง

เมื่อคลิกที่ปุ่ม ส่งการรวมภาษี ไฟล์ XML ของ ประเภทเอกสาร ที่เหมาะสมจะถูกสร้างขึ้น แนบไปกับใบเรียกเก็บเงิน และส่งเป็นใบแจ้งหนี้

ประเภทเอกสาร¶

SdI กำหนดให้ธุรกิจส่งใบแจ้งหนี้ของลูกค้าและเอกสารอื่นๆ ผ่าน EDI

รหัสต่อไปนี้ ประเภทเอกสาร ทั้งหมดจะระบุกรณีการใช้งานทางธุรกิจที่แตกต่างกันในทางเทคนิค

TD01 - ใบแจ้งหนี้¶

นี่แสดงถึงสถานการณ์ ในประเทศ มาตรฐานสำหรับใบแจ้งหนี้ทั้งหมดที่แลกเปลี่ยนผ่าน SdI ใบแจ้งหนี้ใดๆ ที่ไม่เข้าข่ายกรณีพิเศษอย่างใดอย่างหนึ่งจะถูกจัดประเภทเป็นใบแจ้งหนี้ปกติ ซึ่งระบุโดย ประเภทเอกสาร TD01

TD02 - การชำระเงินดาวน์¶

การชำระเงินดาวน์ ใบแจ้งหนี้จะถูกนำเข้า/ส่งออกโดยมีรหัส ประเภทเอกสาร TDO2 แตกต่างจากใบแจ้งหนี้ปกติ เมื่อนำเข้าใบแจ้งหนี้ จะมีการสร้างใบเรียกเก็บเงินของผู้จำหน่ายปกติ

Odoo ส่งออกธุรกรรมเป็น TD02 หากตรงตามเงื่อนไขต่อไปนี้:

  1. เป็นใบแจ้งหนี้;

  2. รายการใบแจ้งหนี้ทั้งหมดเกี่ยวข้องกับรายการใบสั่งขายแบบชำระเงินดาวน์

TD04 - ใบลดหนี้¶

เป็นสถานการณ์มาตรฐานสำหรับ ใบลดหนี้ ทั้งหมดที่ออกให้กับลูกค้า ในประเทศ เมื่อเราจำเป็นต้องรับทราบอย่างเป็นทางการว่าผู้ขายกำลังลดหรือยกเลิกใบแจ้งหนี้ที่ออกก่อนหน้านี้ ตัวอย่างเช่น ในกรณีที่มีการเรียกเก็บเงินเกิน รายการไม่ถูกต้อง หรือการจ่ายเงินมากเกินไป เช่นเดียวกับใบแจ้งหนี้ จะต้องส่งไปที่ SdI ซึ่ง ประเภทเอกสาร TD04

TD07, TD08, TD09 - การออกใบแจ้งหนี้แบบง่าย¶

สามารถใช้ใบแจ้งหนี้แบบง่าย (TD07), ใบลดหนี้ (TD08) และใบเดบิต (TD09) เพื่อรับรองธุรกรรมในประเทศที่มีมูลค่าต่ำกว่า 400 ยูโร (รวมภาษีมูลค่าเพิ่มแล้ว) สถานะจะเหมือนกับใบแจ้งหนี้ทั่วไป แต่มีข้อกำหนดด้านข้อมูลน้อยกว่า

หากต้องการสร้างใบแจ้งหนี้แบบง่าย จะต้องประกอบด้วย:

  1. :guilabel:`การอ้างอิงใบแจ้งหนี้ของลูกค้า: ลำดับหมายเลข ไม่ซ้ำกัน โดย ไม่มีช่องว่าง;

  2. วันที่ในใบแจ้งหนี้: ออก วันที่ ของใบแจ้งหนี้;

  3. ข้อมูลบริษัท: ข้อมูลรับรองทั้งหมดของ ผู้ขาย (หมายเลข VAT/TIN ชื่อ ที่อยู่แบบเต็ม) ภายใต้ การตั้งค่าทั่วไป ‣ บริษัท (ส่วน);

  4. VAT: หมายเลข VAT/TIN ของ ผู้ซื้อ (ในแบบฟอร์มของคู่ค้า)

  5. ทั้งหมด: ยอดรวม จำนวน (รวมภาษีมูลค่าเพิ่ม) ของใบแจ้งหนี้

ใน EDI นั้น Odoo จะส่งออกใบแจ้งหนี้ให้ง่ายขึ้น หาก:

  1. เป็นธุรกรรมในประเทศ (เช่น คู่ค้ามาจากประเทศอิตาลี)

  2. ช่องที่ต้องกรอก ของบริษัทของคุณ (หมายเลขภาษี หรือ Codice Fiscale, ระบบการเงิน และ ที่อยู่ แบบเต็ม) ระบุไว้แล้ว

  3. ที่อยู่ของคู่ค้าไม่ได้ระบุอย่างครบถ้วน (เช่น ไม่มีเมืองหรือรหัสไปรษณีย์)

  4. ยอดรวมภาษีมูลค่าเพิ่มคือ น้อยกว่า กว่า 400 ยูโร

Note

เกณฑ์ 400 ยูโรถูกกำหนดไว้ใน พระราชกฤษฎีกาวันที่ 10 พฤษภาคม 2019 ใน Gazzetta Ufficiale เราแนะนำให้คุณตรวจสอบมูลค่าอย่างเป็นทางการในปัจจุบัน

TD16 - การเรียกเก็บเงินย้อนกลับภายใน¶

ธุรกรรมการเรียกเก็บเงินย้อนกลับภายใน (ดู การยกเว้นภาษี และ การเรียกเก็บเงินย้อนกลับ) จะถูกส่งออกเป็น TD16 หากตรงตามเงื่อนไขต่อไปนี้:

  • เป็นใบเรียกเก็บเงินของผู้ขาย

  • มีอย่างน้อย หนึ่งภาษี บนบรรทัดใบแจ้งหนี้ที่กำหนดเป้าหมายอย่างใดอย่างหนึ่งเหล่านี้ ตารางภาษี: VJ6, VJ7, VJ8, VJ12, VJ13, VJ14, VJ15, VJ16, VJ17

TD17 – การรับซื้อบริการจากต่างประเทศ¶

เมื่อซื้อ บริการ จากประเทศ EU และ นอกสหภาพยุโรป ผู้ขาย ต่างประเทศจะออกใบแจ้งหนี้บริการด้วยราคา ไม่รวม VAT เนื่องจากไม่ต้องเสียภาษีในประเทศอิตาลี VAT ชำระโดย ผู้ซื้อ ในประเทศอิตาลี

  • ภายในสหภาพยุโรป: ผู้ซื้อ รวมใบแจ้งหนี้ที่ได้รับกับ ข้อมูล VAT ที่ต้องชำระในประเทศอิตาลี (เช่น การรวมภาษีการเรียกเก็บเงินของผู้ขาย)

  • นอกสหภาพยุโรป: ผู้ซื้อ ส่งใบแจ้งหนี้ให้ตนเอง (เช่น การเรียกเก็บเงินด้วยตนเอง)

Odoo ส่งออกธุรกรรมเป็น TD17 หากตรงตามเงื่อนไขต่อไปนี้:

  • เป็นใบเรียกเก็บเงินของผู้ขาย

  • มีอย่างน้อย หนึ่งภาษี บนบรรทัดใบแจ้งหนี้ที่กำหนดเป้าหมายตารางภาษี VJ3;

  • รายการใบแจ้งหนี้ทั้งหมดมี บริการ เป็น ผลิตภัณฑ์ หรือภาษีที่มี บริการ เป็น ขอบเขตภาษี

TD18 - การซื้อสินค้าจากสหภาพยุโรป¶

ใบแจ้งหนี้ที่ออกภายในสหภาพยุโรปเป็นไปตาม รูปแบบมาตรฐาน ดังนั้นจึงจำเป็นต้องมีการรวมใบแจ้งหนี้ที่มีอยู่เท่านั้น

Odoo ส่งออกธุรกรรมเป็น TD18 หากตรงตามเงื่อนไขต่อไปนี้:

  • เป็นใบเรียกเก็บเงินของผู้ขาย

  • คู่ค้า มาจาก สหภาพยุโรป

  • มีอย่างน้อยหนึ่งภาษีในรายการใบแจ้งหนี้ที่กำหนดเป้าหมายตารางภาษี VJ9;

  • บรรทัดใบแจ้งหนี้ทั้งหมดมี สินค้าสิ้นเปลือง เป็น ผลิตภัณฑ์ หรือภาษีที่มี สินค้า เป็น ขอบเขตภาษี

TD19 - ซื้อสินค้าจากการฝากภาษีมูลค่าเพิ่ม¶

ซื้อ สินค้า จากผู้จำหน่าย ต่างประเทศ แต่ สินค้า นั้นอยู่ใน ประเทศอิตาลี แล้ว โดยมี ค่ามัดจำ VAT

  • จากสหภาพยุโรป: ผู้ซื้อ รวมใบแจ้งหนี้ที่ได้รับกับ ข้อมูล VAT ที่ต้องชำระในประเทศอิตาลี (เช่น การรวมภาษีการเรียกเก็บเงินของผู้ขาย)

  • นอกสหภาพยุโรป: ผู้ซื้อ ส่งใบแจ้งหนี้ถึง ตนเอง (เช่น การเรียกเก็บเงินด้วยตนเอง)

Odoo ส่งออกธุรกรรมเป็น TD19 หากตรงตามเงื่อนไขต่อไปนี้:

  • เป็นใบเรียกเก็บเงินของผู้ขาย

  • มีอย่างน้อยหนึ่งภาษีในรายการใบแจ้งหนี้ที่กำหนดเป้าหมายตารางภาษี VJ3;

  • บรรทัดใบแจ้งหนี้ทั้งหมดมี สินค้าสิ้นเปลือง เป็นผลิตภัณฑ์หรือภาษีที่มี สินค้า เป็น ขอบเขตภาษี

TD24 - ใบแจ้งหนี้รอการตัดบัญชี¶

ใบแจ้งหนี้รอการตัดบัญชี คือใบแจ้งหนี้ที่ ออกให้ในภายหลัง จากการขายสินค้าหรือการให้บริการ ใบแจ้งหนี้รอการตัดบัญชี จะต้องออกอย่างช้าที่สุดภายใน วันที่ 15 ของเดือนถัดจากการจัดส่งที่ครอบคลุมในเอกสาร

โดยปกติจะเป็น ใบแจ้งหนี้สรุป ซึ่งมีรายการขายสินค้าหรือบริการหลายรายการที่ดำเนินการในเดือนนั้น ธุรกิจได้รับอนุญาตให้ จัดกลุ่ม ยอดขายออกเป็น ใบแจ้งหนี้ใบเดียว ซึ่งโดยทั่วไปจะออกตอน สิ้นเดือน เพื่อวัตถุประสงค์ทางบัญชี ใบแจ้งหนี้ที่เลื่อนออกไปเป็นค่าเริ่มต้นสำหรับ ผู้ค้าส่ง ที่มีลูกค้าประจำ

หากสินค้าถูกขนส่งโดย ผู้ขนส่ง การจัดส่งทุกครั้งจะต้องมี Documento di Transporto (DDT) หรือ เอกสารการขนส่ง ที่เกี่ยวข้อง ใบแจ้งหนี้ที่เลื่อนออกไป ต้อง ระบุรายละเอียดของข้อมูล DDTs ทั้งหมดเพื่อการติดตามที่ดีขึ้น

Note

การออกใบแจ้งหนี้อิเล็กทรอนิกส์ของใบแจ้งหนี้ที่เลื่อนออกไปจำเป็นต้องมี l10n_it_stock_ddt โมดูล ในกรณีนี้ จะมีการใช้ ประเภทเอกสาร TD24 เฉพาะในใบแจ้งหนี้อิเล็กทรอนิกส์

Odoo ส่งออกธุรกรรมเป็น TD24 หากตรงตามเงื่อนไขต่อไปนี้:

  1. เป็นใบแจ้งหนี้;

  2. เกี่ยวข้องกับการจัดส่งที่ DDTs มีวันที่ แตกต่าง จากวันที่ออกใบแจ้งหนี้

TD28 - ซานมารีโน¶

การแจ้งหนี้¶

ซานมารีโนและอิตาลีมีข้อตกลงพิเศษเกี่ยวกับการดำเนินการออกใบแจ้งหนี้อิเล็กทรอนิกส์ ด้วยเหตุนี้ ใบแจ้งหนี้ จึงเป็นไปตามกฎ การเรียกเก็บเงินย้อนกลับ ปกติ คุณสามารถใช้ ประเภทเอกสาร ที่เหมาะสมได้ ทั้งนี้ขึ้นอยู่กับประเภทใบแจ้งหนี้: TD01, TD04, TD05, TD24, TD25 ข้อกำหนดเพิ่มเติมไม่ได้บังคับใช้โดย Odoo อย่างไรก็ตาม รัฐ ร้องขอผู้ใช้ให้:

  • เลือกภาษีโดยตั้งค่า ประเภทการยกเว้นภาษี เป็น N3.3;

  • ใช้คำทั่วไป SdI รหัสปลายทาง 2R4GT08

จากนั้นสำนักงานเฉพาะในซานมารีโนจะส่งใบแจ้งหนี้ไปยังธุรกิจที่ถูกต้อง

บิลผู้ขาย¶

เมื่อได้รับ ใบเรียกเก็บเงินกระดาษ จากซานมารีโน ทุกบริษัทอิตาลี ต้อง ส่งใบแจ้งหนี้นั้นไปที่ AdE โดยระบุ ประเภทเอกสารของใบแจ้งหนี้อิเล็กทรอนิกส์ ` ที่มีค่าพิเศษ `TD28

Odoo ส่งออกธุรกรรมเป็น TD28 หากตรงตามเงื่อนไขต่อไปนี้:

  1. เป็นใบเรียกเก็บเงินของผู้ขาย

  2. มีอย่างน้อยหนึ่งภาษีในรายการใบแจ้งหนี้ที่กำหนดเป้าหมายตารางภาษี VJ;

  3. ประเทศ ของคู่ค้าคือ ซานมารีโน

ธุรกิจบริหารรัฐกิจ (B2G)¶

ธุรกิจ PA อยู่ภายใต้การควบคุมมากกว่าธุรกิจส่วนตัว เนื่องจากต้องจัดการกับเงินสาธารณะที่มาจากผู้เสียภาษี กระบวนการ EDI จะเพิ่มขั้นตอนบางอย่างให้กับ ใบแจ้งหนี้ปกติ เนื่องจากธุรกิจ PA สามารถ ** ยอมรับ** หรือ ปฏิเสธ ใบแจ้งหนี้

Note

PA ธุรกิจมีความยาว 6 หลัก รหัสปลายทาง หรือที่เรียกว่า CUU นั่นคือ บังคับ, **PEC ** ที่อยู่ไม่สามารถใช้งานได้ในกรณีนี้

See also

รายชื่อธุรกิจทั้งหมดที่เป็นของฝ่ายบริหารสาธารณะ พร้อมด้วยรหัสปลายทาง

CIG, CUP, DatiOrdineAcquisto¶

เพื่อให้มั่นใจว่าสามารถตรวจสอบย้อนกลับการชำระเงินโดยหน่วยงานภาครัฐได้อย่างมีประสิทธิภาพ ใบแจ้งหนี้อิเล็กทรอนิกส์ที่ออกให้กับหน่วยงานราชการจะต้องมี:

  • CIG ยกเว้นในกรณีของการยกเว้นจากภาระผูกพันในการตรวจสอบย้อนกลับที่กำหนดโดยกฎหมาย n. 136 วันที่ 13 สิงหาคม 2553;

  • CUP ในกรณีของใบแจ้งหนี้ที่เกี่ยวข้องกับงานสาธารณะ

หากไฟล์ XML จำเป็นต้องใช้ AdE ที่สามารถดำเนินการชำระเงินตามใบแจ้งหนี้อิเล็กทรอนิกส์ได้ เท่านั้น เมื่อไฟล์ XML มี CIG และ :abbr:` CUP (Codice Unico di Progetto)`

Note

CUP และ CIG จะต้องรวมอยู่ในหนึ่งใน DatiOrdineAcquisto, DatiContratto, DatiConvenzione, DateRicezione หรือ แท็ก XML DatiFattureCollegate

สิ่งเหล่านี้สอดคล้องกับองค์ประกอบชื่อ CodiceCUP และ CodiceCIG ของไฟล์ XML ของใบแจ้งหนี้อิเล็กทรอนิกส์ ซึ่งสามารถดูตารางได้บนเว็บไซต์ของรัฐบาล <http://www.fatturapa.gov.it/> `_.

แยกการชำระเงิน¶

กลไก แบ่งการชำระเงิน มีพฤติกรรมเหมือนกับ การเรียกเก็บเงินย้อนกลับ

Example

เมื่อบริษัทอิตาลีเรียกเก็บเงินธุรกิจ PA - ตัวอย่างเช่น บริการทำความสะอาดอาคารสาธารณะ ธุรกิจ PA จะรายงานภาษีมูลค่าเพิ่มด้วยตนเองไปยังหน่วยงานภาษีด้วยตนเอง และผู้ขายเพียงแค่ต้องเลือกภาษีที่เหมาะสมโดยให้สิทธิ์ การยกเว้นภาษี สำหรับรายการใบแจ้งหนี้ของพวกเขา

ตำแหน่งทางการเงินเฉพาะ Scissione dei Pagamenti มีไว้สำหรับจัดการกับคู่ค้าที่เป็นของ PA

กระบวนการ¶

ลายเซ็นดิจิทัลที่ผ่านการรับรอง¶

สำหรับใบแจ้งหนี้และใบเรียกเก็บเงินสำหรับ PA จำเป็นต้องมี ลายเซ็นดิจิทัลที่ผ่านการรับรอง สำหรับไฟล์ทั้งหมดที่ส่งผ่าน SdI ไฟล์ XML จะต้องได้รับการรับรองโดยใช้ใบรับรองที่เป็นอย่างใดอย่างหนึ่งต่อไปนี้

  • สมาร์ทการ์ด;

  • โทเค็น USB;

  • HSM

Warning

Odoo ไม่สามารถ เซ็นเอกสารแบบดิจิทัลให้คุณได้ เมื่อตรวจพบ Codice Destinatario ที่มีความยาว 6 หลัก กระบวนการ EDI จะหยุดลง และใบแจ้งหนี้จะถูกตั้งค่าในสถานะ ต้องมีลายเซ็นผู้ใช้ คุณสามารถดาวน์โหลดเอกสารในรูปแบบ XML ลงนามด้วยโปรแกรมภายนอกของผู้ให้บริการ ลายเซ็นดิจิทัลที่ผ่านการรับรอง ใดก็ได้ และส่งผ่านพอร์ทัล AdE

การยอมรับหรือการปฏิเสธ¶

Warning

เนื่องจาก Odoo ไม่ได้จัดการการส่งใบแจ้งหนี้ที่ลงนามแล้วไปยังธุรกิจ PA Odoo จึงไม่สามารถกระตุ้นสถานะเหล่านี้ได้โดยตรง เมื่อคุณอัปโหลดใบแจ้งหนี้บนพอร์ทัล AdE Odoo จะได้รับการแจ้งเตือนเกี่ยวกับใบแจ้งหนี้ โดยใส่ สถานะ SdI ที่ถูกต้องลงในใบแจ้งหนี้

หลังจากได้รับใบแจ้งหนี้ผ่านทาง SdI ธุรกิจ PA มีเวลา 15 วันในการยอมรับใบแจ้งหนี้ หากเป็นเช่นนั้น กระบวนการก็จะสิ้นสุดที่นี่ หากธุรกิจ PA ปฏิเสธใบแจ้งหนี้ จะถือว่ายังใช้ได้เมื่อได้รับการยอมรับจาก SdI จากนั้นคุณจะต้องออกใบลดหนี้เพื่อชดเชยและส่งไปที่ SdI

เงื่อนไขที่หมดอายุ¶

หากธุรกิจ PA ไม่ตอบกลับภายใน 15 วัน คุณต้องติดต่อธุรกิจ PA โดยตรง โดยส่งทั้งใบแจ้งหนี้และการแจ้งเตือนกำหนดเวลาที่ได้รับภายใน อีเมล. คุณสามารถจัดการกับพวกเขาและตั้งค่า สถานะ SdI ที่ถูกต้องในใบแจ้งหนี้ของคุณได้ด้วยตนเอง
