รายการทำงาน¶

รายการงานถูกสร้างขึ้นโดยอัตโนมัติในแอพ * Payroll * ตามพนักงาน: ref: ประเภทโครงสร้างเงินเดือน <เงินเดือน/โครงสร้างประเภท> และจากการวางแผน *, *การเข้าร่วม *และ *แอปพลิเคชันเวลาหยุด *

The Work Entries dashboard of the Payroll application provides a visual overview of the individual work entries for every employee.

ในการเปิดแดชบอร์ดนำทางไปที่: เมนูการเลือก: `แอพเงินเดือน -> งานรายการ -> รายการทำงาน '

บน: guilabel: dashboard entry intoll, รายการที่ทำงานปรากฏเป็นตัวอักษรคำสั่งซื้อตามชื่อแรกของพนักงาน แสดงด้วยวันปัจจุบันที่เน้นสีเหลืองอ่อน

หากรายการใด ๆ มี: ref: ความขัดแย้ง <เงินเดือน/ความขัดแย้ง> ที่จำเป็นต้องเป็นได้รับการแก้ไขแล้วแผงควบคุมเริ่มต้นเพื่อกรองเฉพาะ: Guilabel: `conflicting`รายการ.

หากต้องการลบตัวกรองออกจาก: Guilabel: ค้นหา ... ` บาร์เพื่อดูงานทั้งหมดรายการ, คลิกที่: guilabel: `✖ (ลบ) ไอคอนบน: Guilabel: `` convatating` filter ใน: guilabel: `search ... ` bar และทั้งหมดรายการงานปรากฏในรายการ

หากต้องการเปลี่ยนมุมมองดังนั้นเฉพาะรายการสำหรับวันเดียวสัปดาห์หรือเดือนเท่านั้นแสดงคลิกที่: Guilabel: Month ตัวเลือกของ: guilabel: day ,: guilabel:` สัปดาห์ 'หรือ: Guilabel: `เดือน' หนึ่งในตัวเลือกในการแสดงข้อมูลสำหรับการเลือกเฉพาะนั้นเท่านั้น

ใช้: Guilabel: ⬅ (ลูกศรซ้าย) และ: Guilabel: ➡ (ลูกศรขวา) ไอคอนทางด้านซ้ายและขวาของ: ปุ่ม Guilabel: `` เดือน 'เพื่อปรับไฟล์วันที่แสดง เลือก

หากรายการงานหายไปและจำเป็นต้องเพิ่มเช่นเวลาป่วยหรือถ้าพนักงานลืมนาฬิกาเข้าและออกเพื่อเปลี่ยนคลิก: Guilabel: on on on: guilabel: `dashboard entry dashboard เพื่อสร้างรายการงานใหม่

ความขัดแย้ง¶

ความขัดแย้งปรากฏขึ้นสำหรับคำขอใด ๆ ที่ยังไม่ได้รับการอนุมัติเช่นป่วยเวลาหรือวันหยุดหรือหากมีข้อผิดพลาดใด ๆ เกี่ยวกับการเข้าทำงานเช่นฟิลด์ที่จำเป็นต้องถูกทิ้งไว้ ก่อนที่จะสามารถสร้างเพย์ลลิปได้

รายการงานใด ๆ ที่มีความขัดแย้งที่จะแก้ไขจะระบุไว้ในหลัก: guilabel: dashboard entry dashboard ซึ่งสามารถเข้าถึงได้โดยการนำทางไปยัง: เมนูการเลือก: `แอพเงินเดือน -> รายการงาน -> รายการทำงาน ' ความขัดแย้งที่ต้องการการแก้ไขจะแสดงโดยค่าเริ่มต้น

ความขัดแย้งจะถูกระบุด้วยสามเหลี่ยมสีส้มที่มุมบนซ้ายของแต่ละรายการทำงาน วันที่และเวลาสำหรับรายการงานเฉพาะจากนั้นคลิก: Guilabel: `แก้ไข 'ดูรายละเอียดความขัดแย้งในหน้าต่างป๊อปอัพ

ความขัดแย้งถูกอธิบายสั้น ๆ ในกล่องข้อความสีส้มใน: Guilabel: `เปิดหน้าต่างป๊อปอัพที่ปรากฏขึ้น

The: Guilabel: Description ,: guilabel:` พนักงาน 'และ: Guilabel: รายการทำงานType อยู่ในรายการด้านซ้ายของหน้าต่างป๊อปอัพ : Guilabel: from และ: Guilabel:` to` ช่วงวันที่และเวลารวมถึงเวลาทั้งหมด (เป็นชั่วโมง) ใน: guilabel: `duration 'ฟิลด์ปรากฏบนด้านขวา.

เวลาปิดความขัดแย้ง¶

ความขัดแย้งในการทำงานที่พบบ่อยที่สุดสำหรับการร้องขอเวลาปิดที่มีถูกส่ง แต่ยังไม่ได้รับการอนุมัติซึ่งส่งผลให้เกิดงานซ้ำกันรายการสำหรับพนักงานคนนั้น (หนึ่งสำหรับเวลาหยุดและอื่น ๆ สำหรับการทำงานปกติ)

หากมีความขัดแย้งเนื่องจากมีการร้องขอเวลาปิดอยู่ในระบบสำหรับเวลาเดียวกับที่รายการงานปกติมีอยู่แล้วการร้องขอเวลาปิดคือป้อนใน: Guilabel: `time Ofl Field '

สร้างผลงานการทำงานใหม่¶

เมื่อมีการสร้างผลงานการทำงานใหม่การเปลี่ยนแปลงใด ๆ ด้วยตนเองเช่นการแก้ไขความขัดแย้งถูกเขียนทับและผลงานจะถูกสร้างใหม่ (หรือสร้างขึ้นใหม่)จากแอปพลิเคชันที่สร้างขึ้น

แนะนำให้แก้ไขวิธีการแก้ไขข้อขัดแย้งจำนวนมากเก็บบันทึกทั้งหมดให้ถูกต้อง ได้รับการแก้ไขเป็นรายบุคคลหากความขัดแย้งเกิดจากสิ่งอื่นแอปพลิเคชันเป็นวิธีปฏิบัติที่ดีที่สุดเพื่อให้แน่ใจว่ามีการบันทึกในอีกด้านหนึ่งแอปพลิเคชันก็ถูกต้อง ความขัดแย้งเหล่านี้ในแอปพลิเคชันที่สร้างความขัดแย้ง

อีกเหตุผลหนึ่งที่แนะนำวิธีนี้ก็คือเมื่อรายการทำงานเป็นการสร้างใหม่ความขัดแย้งจะปรากฏขึ้นอีกครั้งหากปัญหาในแอปพลิเคชันที่เกี่ยวข้องคือ ** ไม่ได้รับการแก้ไข

ขั้นแรกให้ตรวจสอบให้แน่ใจว่าปัญหาได้รับการแก้ไขในแอปพลิเคชันเฉพาะที่ทำให้เกิดความขัดแย้งในการทำงาน

ถัดไปคลิกปุ่ม: guilabel: regenerate entries entries ปุ่มที่ด้านบนของ: Guilabel: dashboard งาน dashboard และ a: guilabel: รายการทำงานหน้าต่างป๊อปอัพ Regeneration` จะปรากฏขึ้น

เลือก: Guilabel: พนักงาน 'เพื่อสร้างผลงานการทำงานใหม่จากเมนูแบบเลื่อนลงและปรับ: guilabel: `from และ: guilabel:` to 'ฟิลด์ดังนั้นช่วงวันที่ที่ถูกต้องจะปรากฏขึ้น

เวลาออกไปรายงาน¶

หากมีการส่งคำขอเวลาปิดเป็นระยะเวลาที่มีอยู่แล้วประมวลผลบน Payslip คำขอปิดเวลาจะปรากฏในหน้า * Time Off *ในแอพ * Payroll * ซึ่งสามารถเข้าถึงได้โดยการนำทางไปยัง: เมนูการเลือก: `แอพเงินเดือน -> รายการงาน -> เวลาปิดการรายงาน '

เพื่อที่จะทำให้เวลาของพนักงานหยุดลงอย่างถูกต้องเวลาปิดคำขอ ** ต้องใช้ ** กับระยะเวลาการจ่ายต่อไปนี้ ทำให้มั่นใจได้ว่ายอดคงเหลือในการร้องขอจะเป็นปัจจุบัน แต่ก็ช่วยลดความต้องการในการทำซ้ำรายการงานยกเลิกการชำระเงินและการออกเงินเดือนใหม่

สถานการณ์ที่พบบ่อยที่สุดเมื่อสถานการณ์นี้เกิดขึ้นคือเมื่อ Payslips เป็นประมวลผลหนึ่งหรือสองวันก่อนที่ระยะเวลาการจ่ายจะสิ้นสุดลงและพนักงานคือป่วยอย่างไม่คาดคิดในวันสุดท้ายของระยะเวลาการจ่ายเงิน ส่งคำขอตามเวลาสำหรับวันที่ประมวลผลไปแล้วในการจ่ายเงินเป็นวันทำงานปกติ รายการและออกใหม่ paycheck Odoo อนุญาตให้เวลาเหล่านั้นหยุดคำขอที่จะนำไปใช้กับระยะเวลาการจ่ายต่อไปนี้แทน
